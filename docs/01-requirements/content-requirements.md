# Content requirements

## Property and positioning

- **REQ-010:** Public copy should reflect a **four-season cottage** in **Haliburton, Ontario**, with emphasis on forest, lake access (e.g. Drag Lake), and comfort (wood stove, indoor/outdoor living).
- **REQ-011:** The home page should orient visitors to **core guides** (welcome, arrival/parking, sleeping, packing, guest access, cooking, nearby) and a **booking** path.

## Required topic areas (guest-facing)

These align with existing or planned posts in the site repo; the list is the requirement set, not an inventory of files.

| Area | Intent |
|------|--------|
| Welcome / overview | Property story and seasonal positioning |
| Getting here and parking | Access, driveway, seasonal notes |
| Sleeping and layout | Beds, spaces, comfort |
| What to bring | Packing vs. what is supplied |
| Guest access and property | Land, dock, boundaries, rules of use |
| Cooking and kitchen | Appliances, BBQ, pantry/meal ideas as needed |
| Local area | Towns, activities, events, hidden gems |
| How-tos | Campfire, wood stove, canoe launch, etc. |
| Seasonal | E.g. winter activities |

## Tone and brand

- **REQ-012:** Welcoming, practical, and accurate—avoid overselling; match guest expectations to reality (steep driveway, weather, etc. where relevant).
- **REQ-013:** English; date and spelling conventions consistent with Canadian context where it matters (e.g. Haliburton, local names).

## What stays off the public site

- **REQ-014:** Lock codes, Wi-Fi passwords, and other **access secrets** must not appear in public Markdown. Use host-to-guest channels (Airbnb, email) for secrets unless a gated area is introduced (not currently in scope).

## Optional expansions

- Dedicated **Contact** and **About** pages if linked from navigation (currently referenced in theme/menu; implementation should match links—see solution backlog).
